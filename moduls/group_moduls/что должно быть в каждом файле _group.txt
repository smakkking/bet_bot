ОБЯЗАТЕЛЬНЫЕ:
-----------------------------------

переменная WALL_URL - содержит в себе ссылку на группу
           NAME - имя группы(сокращенное)

-----------------------------------

переменная BET_TEMPLATES - хранит кортежы из пар (templateN, parseN)
или может присоеденить к себе BET_TEMPLATES из букмекерки(для начала узнать, где ставит каппер)

-----------------------------------

функция check_templates(post, token)
на вход - объект поста и токен для распознавания с пом-ю класса YandexAPI_detection
парсим тут фотки с поста. на фотке может быть соклько угодно ставок
задача - разделить текст так, чтобы его можно было отправить в функции parse()



ДОПОЛНИТЕЛЬНЫЕ:
-----------------------------------
переменная offset_table - таблица смещений для исходов матчей(см примеры в файле готовых групп)
Это словарь, который приводит ставку к общей форме
Общая форма - 
offset_table = {
    # победитель по карте
        'map_winner',
    # фора по карте 
        'map_handicap',
    # тотал по карте
        'map_total',
    # победа команды
        'match_result',
    # фора 
        'handicap',
    # конкретный счет
        'score',
    # тотал(сумма счетов)
        'total',
}

-----------------------------------

функция вида templateN(text) - принимает в качестве агрумента текст с изображения
шаблон для каждой картинки составляется ВРУЧНУЮ!!!!!!
Возвращает True - если текст подъходит под шаблон
Иначе - False

-----------------------------------

функция вида parseN(photo_url, text) - принимает ссылку на картинку и ее текст(в разбитом на слова виде)
Подразумевается, что на вход подается текст, в котором содержится только ОДНА ставка
Возвращает экземпляр класса Stavka()
Как что парсить решать самому(смотреть на варианты текста)
